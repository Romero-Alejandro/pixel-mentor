# AI Recipe Generation - SSE, UI & Types Specification

This document specifies the requirements for implementing a non-blocking AI recipe generation feature using Server-Sent Events (SSE).

---

# Delta for `api/ai`

## ADDED Requirements

### Requirement: AI Recipe Generation SSE Endpoint

The system MUST provide an SSE endpoint for streaming AI-generated recipe steps in real-time.

- **Endpoint**: `GET /api/ai/generate-recipe/stream`
- **Authentication**: The endpoint MUST be protected and require an active user session.

#### Query Parameters

| Parameter | Type     | Description                                       | Required |
| --------- | -------- | ------------------------------------------------- | -------- |
| `topic`   | `string` | The main topic or ingredients for the recipe.     | Yes      |
| `ageMin`  | `number` | The minimum target age for the recipe's audience. | Yes      |
| `ageMax`  | `number` | The maximum target age for the recipe's audience. | Yes      |

#### SSE Event Stream

The endpoint MUST stream events with the following structure:

##### Event: `step`

- **Description**: Sent for each complete recipe step generated by the AI.
- **Data**: A JSON object of type `GeneratedStep` from `@pixel-mentor/shared`.
- **Payload Example**: `data: {"title": "Mix flour and eggs", "description": "Combine the dry and wet ingredients in a bowl."}`

##### Event: `progress`

- **Description**: Sent periodically to update the client on the generation progress.
- **Data**: A JSON object indicating the percentage complete.
- **Payload Example**: `data: {"progress": 50}`

##### Event: `error`

- **Description**: Sent if an error occurs during generation. The connection SHOULD be closed by the server after sending this event.
- **Data**: A JSON object containing an error message.
- **Payload Example**: `data: {"error": "AI service unavailable", "code": 503}`

##### Event: `complete`

- **Description**: Sent when the recipe generation is fully complete. The connection MUST be closed by the server after this event.
- **Data**: A JSON object confirming completion.
- **Payload Example**: `data: {"status": "done"}`

#### Scenario: Successful Recipe Generation

- **GIVEN** a user is authenticated
- **AND** the user sends a request to `GET /api/ai/generate-recipe/stream` with valid `topic`, `ageMin`, and `ageMax` query parameters
- **WHEN** the backend AI service generates recipe steps
- **THEN** the server MUST send one or more `step` events, each containing a `GeneratedStep` object.
- **AND** the server MAY send `progress` events.
- **AND** the server MUST send a `complete` event at the end of the stream.
- **AND** the server MUST close the connection.

#### Scenario: AI Service Error

- **GIVEN** a user is connected to the generation stream
- **WHEN** the backend AI service fails
- **THEN** the server MUST send an `error` event with a descriptive message.
- **AND** the server MUST close the connection.

# Delta for `web/recipe-generation`

## MODIFIED Requirements

### Requirement: Non-Blocking Recipe Generation UI

The AI recipe generation UI MUST be updated to use the SSE stream, remaining responsive and providing real-time feedback to the user.

#### Scenario: Initiating Recipe Generation

- **GIVEN** the user is on the recipe creation page
- **WHEN** the user clicks the "Generate with AI" button
- **THEN** the "Generate with AI" button MUST be disabled.
- **AND** a loading indicator or skeleton UI MUST be displayed in the area where steps will appear.
- **AND** a progress indicator (e.g., a progress bar) SHOULD be displayed, initialized to 0%.

#### Scenario: Receiving Recipe Steps

- **GIVEN** the generation process is active
- **WHEN** the frontend receives a `step` event from the SSE stream
- **THEN** the new recipe step MUST be appended to the list of generated steps in the UI.
- **AND** the UI SHOULD smoothly animate the new step into view.

#### Scenario: Receiving Progress Updates

- **GIVEN** the generation process is active
- **WHEN** the frontend receives a `progress` event from the SSE stream
- **THEN** the progress indicator's value MUST be updated to reflect the new progress percentage.

#### Scenario: Finalizing Generation

- **GIVEN** the generation process is active
- **WHEN** the frontend receives the `complete` event from the SSE stream
- **THEN** the loading indicator MUST be hidden.
- **AND** the progress indicator SHOULD be hidden or show 100%.
- **AND** the "Generate with AI" button MAY be re-enabled for another run.

### Requirement: Asynchronous "Apply Steps" UI

The UI for applying generated steps to the main recipe form MUST provide feedback while the operation is in progress.

#### Scenario: Applying Generated Steps

- **GIVEN** a list of AI-generated steps is displayed
- **WHEN** the user clicks the "Apply" button
- **THEN** the "Apply" button MUST be disabled.
- **AND** the button's text SHOULD change to "Applying..." or show a spinner.
- **AND** once the steps are successfully added to the main recipe store, the generated steps UI SHOULD be cleared and the button returned to its original state.

# Delta for `shared/types`

## MODIFIED Requirements

### Requirement: Centralized Frontend Types

The frontend application MUST use the canonical data types from the `@pixel-mentor/shared` package for AI-generated recipe content.

#### Scenario: Importing Shared Types

- **GIVEN** a frontend component or store needs to handle recipe data (e.g., `recipe-step-ai-generator.tsx`)
- **WHEN** defining types for variables or function signatures
- **THEN** types like `GeneratedStep` MUST be imported from `@pixel-mentor/shared/src/types/recipe.ts`.

## REMOVED Requirements

### Requirement: Local Frontend Type Definitions

Any local `interface` or `type` definitions in the `apps/web` project that duplicate types from `@pixel-mentor/shared` MUST be removed.

- **File Example**: `apps/web/src/app/recipes/components/recipe-step-ai-generator.tsx`
- **Action**: Remove any declaration of `interface GeneratedStep { ... }`.
